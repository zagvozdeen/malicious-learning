Различные схемы кросс-валидации нужны в разных ситуациях. Например, **перемешанная K-fold валидация** (разбить данные на _k_ частей) подходит в общем случае для оценки модели. **Stratified K-fold** используется в классификации с несбалансированными классами - она следит, чтобы в каждом фолде доля классов сохранялась, это пригодится, когда один класс редкий. **Leave-One-Out (LOOCV)** может быть полезен при очень малом объёме данных: модель каждый раз учится на всех данных кроме одного примера, так максимально используются данные. А для временных рядов применяют специальную **скользящую валидацию по времени** - когда данные разбиваются по хронологии (не перемешиваются), чтобы моделировать прогнозирование будущего из прошлого. В каждой из этих ситуаций свой метод кросс-валидации помогает более корректно оценить модель.
